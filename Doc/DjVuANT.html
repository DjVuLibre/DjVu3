Documentation

This class contains some trivial annotations of the page or of the document such as page border color, page alignment, initial zoom and display mode, hyperlinks and highlighted areas. All this information is put inside a textual chunk ANTa in pseudo-lisp format. Decoding and encoding are normally done by decode() and encode() functions.
DjVuANT(void)
Constructs an empty annotation object.

static GP<DjVuANT> create(void)
Creates an empty annotation object.

unsigned long int bg_color
Background color. Is in 0x00RRBBGG format. 0xffffffff if there were no background color records in the annotation chunk.

int zoom
Initial zoom. Possible values are:
ZOOM_STRETCH
the image is stretched to the viewport.
ZOOM_ONE2ONE
the image is displayed pixel-to-pixel.
ZOOM_WIDTH
"Fit width" mode.
ZOOM_PAGE
"Fit page" mode.
ZOOM_UNSPEC
Annotation does not specify a zoom factor.

int mode
Initial mode. Possible values are:
MODE_COLOR
color mode.
MODE_FORE
foreground mode.
MODE_BACK
background mode.
MODE_BW
black and white mode.
MODE_UNSPEC
Annotation does not specify a display mode.
Any positive number
Zoom in %. Please note that all constants above are either negative or ZERO. Thus it's possible to distinguish numerical zoom from those special cases.

int hor_align
Horizontal page alignment. Possible values are ALIGN_LEFT, ALIGN_CENTER, ALIGN_RIGHT and ALIGN_UNSPEC.

int ver_align
Vertical page alignment. Possible values are ALIGN_TOP, ALIGN_CENTER, ALIGN_BOTTOM and ALIGN_UNSPEC.

GPList<GMapArea> map_areas
List of defined map areas. They may be just areas of highlighting or hyperlink. Please refer to GMapArea, GMapRect, GMapPoly and GMapOval for details.

bool is_empty(void) const
Returns TRUE if no features are specified or specified features are not different from default ones

void decode(ByteStream &bs)
Decodes contents of annotation chunk ANTa. The chunk data is read from ByteStream bs until reaching an end-of-stream marker. This function is normally called after a call to get_chunk().

void merge(ByteStream & bs)
Same as decode() but adds the new data to what has been decoded before.

void encode(ByteStream &bs)
Encodes the ANTa chunk. The annotation data is simply written into ByteStream bs with no IFF header. This function is normally called after a call to put_chunk().

GUTF8String encode_raw(void) const
Encodes data back into raw annotation data.

Returns a copy of this object

unsigned int get_memory_usage() const
Returns the number of bytes needed by this data structure. It's used by caching routines to estimate the size of a DjVuImage.
