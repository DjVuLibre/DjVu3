HOW TO BUILD gcc-2.95.3

Step 1: Optain a binary working version of gcc.  This can be any version
        greater than or equal to 2.8.

        - I know it is silly.  But it is the chicken and egg problem.
        - There are many places you can find gcc on the net for download,
          in binary format, copy gcc off another machine, build gcc on
          a previous version of the same architecture, or build with a cross compiler.

Step 2: Download binutils-2.11.tar.gz and gcc-2.95.3.tar.gz from www.gnu.org.

Step 3: Unzip these.  If you don't have gunzip, you can unzip them on
        different box, and then copy them over.
                gunzip *.tar.gz

Step 4. Untar the files.  i.e.
            cd $HOME
            tar xvvf binutils-2.11.tar
            tar xvvf gcc-2.95.3.tar

Step 5. Configure binutils-2.11 for a directory not currently in your
        search path:
            mkdir $HOME/gcc-build
            cd ~/binutils*11;./configure --prefix=$HOME/gcc-build

Step 6.  Build and install binutils-2.11:
            make
            make install

Step 7.  Add this new path to your PATH variable:
            export PATH=$HOME/gcc-build/bin:$PATH

Step 8.  Configure gcc:
            cd $HOME/gcc*3
            ./configure --prefix=$HOME/gcc-build \
               --with-gnu-as --with-gnu-ld --with-threads

Step 9.  Build and install gcc:
            make bootstrap
            make install

Step 10. Remove your build directories and extract again:
            cd $HOME
            rm -rf gcc*3 binutils*11
            tar xvvf binutils-2.11.tar
            tar xvvf gcc-2.95.3.tar

Step 11. Reconfigure the binutils:
            cd ~/binutils*11;./configure --prefix=/usr/local

Step 12. Build and install binutils-2.11:
            make
            su -c "make install"

Step 13. Make sure /usr/local/bin is first in your search path:
            export PATH=/usr/local/bin:$PATH

Step 14. Configure gcc:
            cd $HOME/gcc*3
            ./configure --prefix=/usr/local \
               --with-gnu-as --with-gnu-ld --with-threads

Step 15. Build and install gcc.
            make bootstrap

            su -c "make install"

Step 16. Remove your first build of gcc, as you don't need it anymore:
            rm -rf $HOME/gcc-build

You now have a version of gcc-2.95.3 which is correctly built.  You
can transport this to any other  machine of the same OS version
without repeating steps 1-16 by simply:
      On the machine you just built gcc:
        tar cvvf $HOME/gcc.tar /usr/local
      On the next machine you wish to use gcc on:
        cd /;tar xvvf $HOME/gcc.tar
